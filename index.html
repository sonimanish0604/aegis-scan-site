How it works

Send a PDF via multipart POST. Receive a structured JSON verdict you can enforce in content gateways, intake forms, or data privacy workflows.

1) Upload

POST /analyze-pdf with your API key and file.

2) Inspect

Engine parses structure and inspects flags without retaining file content.

3) Score

Risk score + granular flags help automated routing and user prompts.

4) Act

Block, quarantine, request a clean copy, or continue with redaction/CDR.

API

Minimal REST interface. Authenticate with X-API-Key. Request up to 25 MB per file (configurable).

Endpoint

POST /analyze-pdf
Headers:
  accept: application/json
  X-API-Key: <YOUR_KEY>
Multipart Form:
  file: @/path/to/file.pdf

cURL

curl -X POST "https://YOUR-RENDER-URL/analyze-pdf" \
  -H "accept: application/json" \
  -H "X-API-Key: YOUR_KEY" \
  -F "file=@/path/to/sample.pdf"
